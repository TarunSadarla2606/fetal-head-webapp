Clinical Workflow
Step 01
Image Acquisition
Upload DICOM or JPEG from your ultrasound system. Standard axial HC plane per ISUOG 2020.
Step 02
AI Segmentation
Residual U-Net delineates the fetal skull boundary. Pixel spacing corrects output to real-world mm.
Step 03
Clinical Report
LLM synthesises HC, GA estimate, growth centile, and confidence into a physician-ready narrative.
Step 04
EHR / PACS Export
FHIR R4 Observation or DICOM SR pushed to your PACS viewer or EHR system.
Clinical Value
  • Reduces manual HC measurement time from ~3 min to under 10 seconds per frame
  • Sub-2mm accuracy meets ISUOG Doppler-equivalent precision thresholds
  • Four model variants span full workstations to bedside tablets (Edge models −43% params)
  • GradCAM++ saliency overlay lets clinicians audit AI attention before accepting measurement
  • Temporal model supports cine-loop analysis for dynamic acquisitions
  • Trained on HC18 (Radboud UMC) — prospectively validated multi-centre dataset
Integration Pathway
  • DICOM C-STORE listener receives from any IHE-compliant ultrasound system
  • Pixel spacing from tag (0028,0030) calibrates output to real-world millimetres
  • Results encoded as FHIR R4 Observation (LOINC 11779-6) for EHR ingestion
  • DICOM SR wraps segmentation mask for PACS viewer overlay
  • REST inference endpoint via TorchServe or BentoML container
  • HL7 v2 ORU fallback for legacy RIS/PACS environments

DICOM / PACS Integration
IHE-compliant pipeline for production deployment
1
DICOM C-STORE Receiver
AE: FETALSCAN_AI · Port 11112
Any IHE-compliant ultrasound system pushes images to the FetalScan listener via standard DICOM C-STORE.
2
Pixel Spacing Extraction
tag (0028,0030) → mm/pixel
Calibration factor extracted from the DICOM header converts segmentation output to real-world millimetres.
3
TorchServe Inference
POST /v2/models/fetalscan/infer
Containerised model endpoint processes the image and returns HC measurement, mask, and confidence score.
4
DICOM Structured Report
SOP 1.2.840.10008.5.1.4.1.1.88.22
Results encoded as DICOM SR with segmentation mask and sent back to PACS for viewer integration.
5
PACS Viewer Display
WADO-RS / DICOMweb pull
Clinician sees HC overlay in their existing PACS viewer without any workflow change.
FHIR R4 / EHR Integration
Structured data for EHR ingestion and longitudinal tracking
1
Observation Resource
LOINC 11779-6 · Head circumference

2

For legacy RIS/PACS without FHIR support, an HL7 v2.x ORU message is generated automatically.
Out-of-Distribution (OOD) Safety Framework
Automatic image quality and distribution checks before measurement is accepted
🔬
Input Validation
validate_input() in inference.py checks image dimensions, bit depth, and intensity histogram before the model runs. Rejects clearly non-ultrasound images.
📊
MC-Dropout Uncertainty
Epistemic uncertainty estimated via Monte Carlo dropout (T=20 passes). Measurements with uncertainty >0.12 are flagged as low-confidence and displayed in amber.
🗺️
GradCAM++ Audit
Saliency heatmap lets the reviewing clinician confirm AI attention is concentrated on the calvarium. Diffuse or peripheral attention suggests a non-standard plane.
✈️
Plane Classifier (Roadmap)
A secondary classifier to auto-detect standard vs. non-standard HC plane will be added in v3.0. Currently relies on clinician verification of image plane before upload.
⚖️
Confidence Threshold
Segmentation confidence <85% triggers a “review required” flag. Measurement is shown with a yellow warning prompting manual verification.
📋
Audit Logging
Every inference is logged with model version, input hash, confidence score, and OOD flag to a tamper-evident audit store for regulatory traceability.
